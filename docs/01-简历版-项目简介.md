# 01 · 简历版项目简介

> 用途:直接用于秋招 / 社招简历"项目经历"栏。
> 原则:**一句话说清做了什么 + 量化结果 + 区别于 CRUD 的技术亮点**。LLM 助手是杀手锏,放在显眼位置。

---

## 一句话定位

> 基于 **Spring Boot 3 + Redis + Redisson** 的新能源汽车充电桩**智能调度平台**,在高并发防超卖地基之上,自研**四层智能化架构**(智能匹配 / 预测定价 / 实时推送 / LLM 自然语言助手),把"被动管理的充电 CRUD"升级为"主动决策的调度系统"。

---

## 简历项目卡(可直接复制)

**PowerQueue · 新能源充电桩智能调度平台** ｜ Java 后端 ｜ 个人/团队项目

- **技术栈**:Java 17、Spring Boot 3.2、Spring MVC、MyBatis-Plus、MySQL 8、Redis、Redisson、WebSocket、Resilience4j、DeepSeek LLM API、JWT;前端 Vue3 + Element Plus + ECharts。

- **高并发抢桩防超卖**:设计「Redis 缓存降压 + Redisson 分布式锁 + DB 乐观锁(`version`)」**三道防线**,核心更新 `UPDATE ... WHERE status='IDLE' AND version=?` 按影响行数判定,**锁在事务提交前释放也不超卖**;`ReservationConcurrencyTest` 验证 **20 并发抢 1 桩仅 1 成功**。

- **L1 智能匹配推荐引擎**:用户只需「我要充电」,系统按
  `score = w1×(1/距离) + w2×(1/等待时间) + w3×(1/价格) + w4×功率匹配度`
  多因子加权输出 Top-N 充电桩;接入高德/百度地图算真实行驶距离,权重后台可配置;各因子归一化到 [0,1] 后加权。

- **L1 等待队列 + 等待时长预估**:桩状态扩展为 `[IDLE, RESERVED, CHARGING, FAULT, QUEUED]`,满桩不再直接报错,而是用 **Redis Sorted Set** 入队(`score=入队时间`),`预估等待 = 排队人数 × 单桩平均充电时长`,轮到时 **WebSocket 推送 + 10 分钟确认窗口**,**淘汰前端 5 秒轮询**。

- **L2 动态供需定价**:落地 `最终电价 = 基础电价 × 时段系数 × 站点负载系数`(谷时 0–8 点 **0.7**、峰时 17–21 点 **1.5**;空闲率 >50% **0.9** 引流、<20% **1.3** 分流),用价格杠杆自趋供需均衡,并在阈值边界加**滞后区间**防价格震荡。

- **L2 分时段需求预测**:按「站点 + 小时 + 星期」统计历史占用率,用**移动平均 + 季节性分解**提前标记负荷,前端 🟢🟡🔴 三色展示(轻量化算法,无深度学习)。

- **L3 实时体验**:WebSocket **事件驱动全量推送**(桩状态变更 → 后端广播 → 前端**增量局部刷新**);充电计划向导基于用户画像 + 规则引擎 + 定时任务实现周期自动预约与漏充提醒。

- **L4 LLM 自然语言助手(杀手锏)**:接入 **DeepSeek**,用户口语输入「我人在国贸,着急,去哪最快充上」,后端**两阶段编排**——先做意图/槽位提取,再**串联平台实时接口**(地图算路 → 匹配推荐 → 等待预估 → 动态定价)拿真实数据,**模型只组织语言不产生数据**,从架构上杜绝幻觉;封装统一 SDK 加**超时/熔断/降级**,LLM 异常自动切回智能匹配页。

---

## 核心技术亮点(精简 bullet,挑 5 条贴简历)

1. **三道防线防超卖**:分布式锁串行化 + 乐观锁兜底 + 缓存降压,20 并发抢 1 桩 0 超卖。
2. **多因子推荐打分**:距离/等待/价格/功率匹配度四因子加权,Top-N 智能撮合,替代手动翻列表。
3. **Redis Sorted Set 等待队列**:满桩可排队、等待时长可预估、轮到自动推送,**优于纯分布式锁**的调度能力。
4. **价格杠杆动态定价**:时段 × 负载 双系数,引流分流自均衡,带滞后区防震荡。
5. **LLM 多接口编排 + 容错**:DeepSeek 两阶段链路,真实数据注入防幻觉,Resilience4j 熔断降级保稳定。

## 量化成果(可按实际压测数据替换)

| 维度 | 成果 |
|------|------|
| 并发安全 | 20 线程抢同一桩,**仅 1 成功、0 超卖**(单元测试断言) |
| 实时性 | 轮询 → WebSocket 推送,状态刷新**延迟从 5s 降至秒级**,接口 QPS 下降(省掉无效轮询) |
| 读压力 | 热门站点状态走 Redis 缓存,**MySQL 读 QPS 显著降低** |
| 供需均衡 | 动态定价上线后,高峰站排队人数向低谷站转移(分流效果,按实际埋点填数) |
| 差异化 | LLM 助手为面试核心记忆点,区别于普通 CRUD 项目 |

> ⚠️ 简历纪律:未真实压测的数字写"显著降低 / 秒级",**不要编造精确 QPS**;一旦写具体数值(如 5000 QPS),面试必被追问压测工具 / 机器配置,答不上反而减分。
